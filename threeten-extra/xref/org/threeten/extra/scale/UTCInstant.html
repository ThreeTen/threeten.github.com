40  /**
41   * An instantaneous point on the time-line measured in the UTC time-scale
42   * with leap seconds.
43   * <p>
44   * The <code>java.time</code> classes use the Java time-scale for simplicity.
45   * That scale works on the assumption that the time-line is simple, there are no leap-seconds
46   * and there are always 24 * 60 * 60 seconds in a day. Unfortunately, the Earth's rotation
47   * is not straightforward, and a solar day does not match this definition.
48   * <p>
49   * This class is an alternative representation based on the UTC time-scale which
50   * includes leap-seconds. Leap-seconds are additional seconds that are inserted into the
51   * year-month-day-hour-minute-second time-line in order to keep UTC in line with the solar day.
52   * When a leap second occurs, an accurate clock will show the time {@code 23:59:60} just before midnight.
53   * <p>
54   * Leap-seconds are announced in advance, typically at least six months.
55   * The {@link UTCRules} class models which dates have leap-seconds.
56   * Alternative implementations of the rules may be supplied.
57   * <p>
58   * The default rules implementation fixes the start point of UTC as 1972.
59   * This date was chosen as UTC was more complex before 1972.
60   * <p>
61   * The duration between two points on the UTC time-scale is calculated solely using this class.
62   * Do not use the {@code between} method on {@code Duration} as that will lose information.
63   * Instead use {@link #durationUntil(UTCInstant)} on this class.
64   * <p>
65   * It is intended that most applications will use the {@code Instant} class
66   * which uses the UTC-SLS mapping from UTC to guarantee 86400 seconds per day.
67   * Specialist applications with access to an accurate time-source may find this class useful.
68   *
69   * <h3>Time-scale</h3>
70   * <p>
71   * The length of the solar day is the standard way that humans measure time.
72   * As the Earth's rotation changes, the length of the day varies.
73   * In general, a solar day is slightly longer than 86400 SI seconds.
74   * The actual length is not predictable and can only be determined by measurement.
75   * The UT1 time-scale captures these measurements.
76   * <p>
77   * The UTC time-scale is a standard approach to bundle up all the additional fractions
78   * of a second from UT1 into whole seconds, known as <i>leap-seconds</i>.
79   * A leap-second may be added or removed depending on the Earth's rotational changes.
80   * If it is removed, then the relevant date will have no time of {@code 23:59:59}.
81   * If it is added, then the relevant date will have an extra second of {@code 23:59:60}.
82   * <p>
83   * The modern UTC time-scale was introduced in 1972, introducing the concept of whole leap-seconds.
84   * Between 1958 and 1972, the definition of UTC was complex, with minor sub-second leaps and
85   * alterations to the length of the notional second.
86   * <p>
87   * This class may be used for instants in the far past and far future.
88   * Since some instants will be prior to 1972, it is not strictly an implementation of UTC.
89   * Instead, it is a proleptic time-scale based on TAI and equivalent to it since 1972.
90   *
91   * <h3>Implementation Requirements:</h3>
92   * This class is immutable and thread-safe.
93   * <p>
94   * This class must be treated as a value type. Do not synchronize, rely on the
95   * identity hash code or use the distinction between equals() and ==.
96   */

131      * Obtains an instance of {@code UTCInstant} from a Modified Julian Day with
132      * a nanosecond fraction of second using the system default leap second rules.
133      * <p>
134      * This factory creates an instance of a UTC instant.
135      * The nanosecond of day value includes any leap second and has a valid range from
136      * {@code 0} to {@code 86,400,000,000,000 - 1} on days other than leap-second-days
137      * and other lengths on leap-second-days.
138      * <p>
139      * The nanosecond value must be positive even for negative values of Modified
140      * Julian Days. One nanosecond before Modified Julian Day zero will be
141      * {@code -1} days and the maximum nanosecond value.
142      *
143      * @param mjDay  the date as a Modified Julian Day (number of days from the epoch of 1858-11-17)
144      * @param nanoOfDay  the nanoseconds within the day, including leap seconds
145      * @return the UTC instant, not null
146      * @throws IllegalArgumentException if nanoOfDay is out of range
147      */
148     public static UTCInstant ofModifiedJulianDay(long mjDay, long nanoOfDay) {
149         return ofModifiedJulianDay(mjDay, nanoOfDay, UTCRules.system());
150     }

152     /**
153      * Obtains an instance of {@code UTCInstant} from a Modified Julian Day with
154      * a nanosecond fraction of second using the specified leap second rules.
155      * <p>
156      * This factory creates an instance of a UTC instant.
157      * The nanosecond of day value includes any leap second and has a valid range from
158      * {@code 0} to {@code 86,400,000,000,000 - 1} on days other than leap-second-days
159      * and other lengths on leap-second-days.
160      * <p>
161      * The nanosecond value must be positive even for negative values of Modified
162      * Julian Days. One nanosecond before Modified Julian Day zero will be
163      * {@code -1} days and the maximum nanosecond value.
164      *
165      * @param mjDay  the date as a Modified Julian Day (number of days from the epoch of 1858-11-17)
166      * @param nanoOfDay  the nanoseconds within the day, including leap seconds
167      * @param rules  the UTC rules to use, not null
168      * @return the UTC instant, not null
169      * @throws IllegalArgumentException if nanoOfDay is out of range
170      */
171     public static UTCInstant ofModifiedJulianDay(long mjDay, long nanoOfDay, UTCRules rules) {
172         long leapSecs = rules.getLeapSecondAdjustment(mjDay);
173         long maxNanos = (SECS_PER_DAY + leapSecs) * NANOS_PER_SECOND;
174         if (nanoOfDay < 0 || nanoOfDay >= maxNanos) {
175             throw new IllegalArgumentException("Nanosecond-of-day must be between 0 and " + maxNanos + " on date " + mjDay);
176         }
177         return new UTCInstant(mjDay, nanoOfDay, rules);
178     }

454     /**
455      * Checks if this instant is equal to the specified {@code UTCInstant}.
456      * <p>
457      * The comparison is based on the positions on the time-line and the rules.
458      * This definition means that two instants representing the same instant on

465     @Override
466     public boolean equals(Object otherInstant) {
467         if (this == otherInstant) {
468             return true;
469         }
470         if (otherInstant instanceof UTCInstant) {
471             UTCInstant other = (UTCInstant) otherInstant;
472             return this.mjDay == other.mjDay &&
473                    this.nanoOfDay == other.nanoOfDay &&
474                    this.rules.equals(other.rules);
475         }
476         return false;
477     }

491     //-----------------------------------------------------------------------
492     /**
493      * A string representation of this instant.
494      * <p>
495      * The string is formatted using ISO-8601.
496      *
497      * @return a representation of this instant, not null
498      */
499     @Override
500     public String toString() {
501         LocalDate date = LocalDate.MAX.with(JulianFields.MODIFIED_JULIAN_DAY, mjDay);  // TODO: capacity/import issues
502         StringBuilder buf = new StringBuilder(18);
503         int sod = (int) (nanoOfDay / NANOS_PER_SECOND);
504         int hourValue = sod / (60 * 60);
505         int minuteValue = (sod / 60) % 60;
506         int secondValue = sod % 60;
507         if (hourValue == 24) {
508             hourValue = 23;
509             minuteValue = 59;
510             secondValue += 60;
511         }
512         int nanoValue = (int) (nanoOfDay % NANOS_PER_SECOND);
513         buf.append(date).append('T')
514             .append(hourValue < 10 ? "0" : "").append(hourValue)
515             .append(minuteValue < 10 ? ":0" : ":").append(minuteValue)
516             .append(secondValue < 10 ? ":0" : ":").append(secondValue);
517         int pos = buf.length();
518         buf.append(nanoValue + NANOS_PER_SECOND);
519         buf.setCharAt(pos, '.');
520         buf.append("(UTC)");
521         return buf.toString();
522     }
