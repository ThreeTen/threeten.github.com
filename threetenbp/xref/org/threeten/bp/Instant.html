69  /**
70   * An instantaneous point on the time-line.
71   * <p>
72   * This class models a single instantaneous point on the time-line.
73   * This might be used to record event time-stamps in the application.
74   * <p>
75   * For practicality, the instant is stored with some constraints.
76   * The measurable time-line is restricted to the number of seconds that can be held
77   * in a {@code long}. This is greater than the current estimated age of the universe.
78   * The instant is stored to nanosecond resolution.
79   * <p>
80   * The range of an instant requires the storage of a number larger than a {@code long}.
81   * To achieve this, the class stores a {@code long} representing epoch-seconds and an
82   * {@code int} representing nanosecond-of-second, which will always be between 0 and 999,999,999.
83   * The epoch-seconds are measured from the standard Java epoch of {@code 1970-01-01T00:00:00Z}
84   * where instants after the epoch have positive values, and earlier instants have negative values.
85   * For both the epoch-second and nanosecond parts, a larger value is always later on the time-line
86   * than a smaller value.
87   *
88   * <h3>Time-scale</h3>
89   * <p>
90   * The length of the solar day is the standard way that humans measure time.
91   * This has traditionally been subdivided into 24 hours of 60 minutes of 60 seconds,
92   * forming a 86400 second day.
93   * <p>
94   * Modern timekeeping is based on atomic clocks which precisely define an SI second
95   * relative to the transitions of a Caesium atom. The length of an SI second was defined
96   * to be very close to the 86400th fraction of a day.
97   * <p>
98   * Unfortunately, as the Earth rotates the length of the day varies.
99   * In addition, over time the average length of the day is getting longer as the Earth slows.
100  * As a result, the length of a solar day in 2012 is slightly longer than 86400 SI seconds.
101  * The actual length of any given day and the amount by which the Earth is slowing
102  * are not predictable and can only be determined by measurement.
103  * The UT1 time-scale captures the accurate length of day, but is only available some
104  * time after the day has completed.
105  * <p>
106  * The UTC time-scale is a standard approach to bundle up all the additional fractions
107  * of a second from UT1 into whole seconds, known as <i>leap-seconds</i>.
108  * A leap-second may be added or removed depending on the Earth's rotational changes.
109  * As such, UTC permits a day to have 86399 SI seconds or 86401 SI seconds where
110  * necessary in order to keep the day aligned with the Sun.
111  * <p>
112  * The modern UTC time-scale was introduced in 1972, introducing the concept of whole leap-seconds.
113  * Between 1958 and 1972, the definition of UTC was complex, with minor sub-second leaps and
114  * alterations to the length of the notional second. As of 2012, discussions are underway
115  * to change the definition of UTC again, with the potential to remove leap seconds or
116  * introduce other changes.
117  * <p>
118  * Given the complexity of accurate timekeeping described above, this Java API defines
119  * its own time-scale with a simplification. The Java time-scale is defined as follows:
120  * <p><ul>
121  * <li>midday will always be exactly as defined by the agreed international civil time</li>
122  * <li>other times during the day will be broadly in line with the agreed international civil time</li>
123  * <li>the day will be divided into exactly 86400 subdivisions, referred to as "seconds"</li>
124  * <li>the Java "second" may differ from an SI second</li>
125  * </ul><p>
126  * Agreed international civil time is the base time-scale agreed by international convention,
127  * which in 2012 is UTC (with leap-seconds).
128  * <p>
129  * In 2012, the definition of the Java time-scale is the same as UTC for all days except
130  * those where a leap-second occurs. On days where a leap-second does occur, the time-scale
131  * effectively eliminates the leap-second, maintaining the fiction of 86400 seconds in the day.
132  * <p>
133  * The main benefit of always dividing the day into 86400 subdivisions is that it matches the
134  * expectations of most users of the API. The alternative is to force every user to understand
135  * what a leap second is and to force them to have special logic to handle them.
136  * Most applications do not have access to a clock that is accurate enough to record leap-seconds.
137  * Most applications also do not have a problem with a second being a very small amount longer or
138  * shorter than a real SI second during a leap-second.
139  * <p>
140  * If an application does have access to an accurate clock that reports leap-seconds, then the
141  * recommended technique to implement the Java time-scale is to use the UTC-SLS convention.
142  * <a href="http://www.cl.cam.ac.uk/~mgk25/time/utc-sls/">UTC-SLS</a> effectively smoothes the
143  * leap-second over the last 1000 seconds of the day, making each of the last 1000 "seconds"
144  * 1/1000th longer or shorter than a real SI second.
145  * <p>
146  * One final problem is the definition of the agreed international civil time before the
147  * introduction of modern UTC in 1972. This includes the Java epoch of {@code 1970-01-01}.
148  * It is intended that instants before 1972 be interpreted based on the solar day divided
149  * into 86400 subdivisions.
150  * <p>
151  * The Java time-scale is used by all date-time classes.
152  * This includes {@code Instant}, {@code LocalDate}, {@code LocalTime}, {@code OffsetDateTime},
153  * {@code ZonedDateTime} and {@code Duration}.
154  *
155  * <h3>Specification for implementors</h3>
156  * This class is immutable and thread-safe.
157  */

272     /**
273      * Obtains an instance of {@code Instant} using seconds from the
274      * epoch of 1970-01-01T00:00:00Z and nanosecond fraction of second.
275      * <p>
276      * This method allows an arbitrary number of nanoseconds to be passed in.
277      * The factory will alter the values of the second and nanosecond in order
278      * to ensure that the stored nanosecond is in the range 0 to 999,999,999.
279      * For example, the following will result in the exactly the same instant:
280      * <pre>
281      *  Instant.ofSeconds(3, 1);
282      *  Instant.ofSeconds(4, -999_999_999);
283      *  Instant.ofSeconds(2, 1000_000_001);
284      * </pre>
285      *
286      * @param epochSecond  the number of seconds from 1970-01-01T00:00:00Z
287      * @param nanoAdjustment  the nanosecond adjustment to the number of seconds, positive or negative
288      * @return an instant, not null
289      * @throws DateTimeException if the instant exceeds the maximum or minimum instant
290      * @throws ArithmeticException if numeric overflow occurs
291      */
292     public static Instant ofEpochSecond(long epochSecond, long nanoAdjustment) {
293         long secs = Jdk8Methods.safeAdd(epochSecond, Jdk8Methods.floorDiv(nanoAdjustment, NANOS_PER_SECOND));
294         int nos = Jdk8Methods.floorMod(nanoAdjustment, NANOS_PER_SECOND);
295         return create(secs, nos);
296     }

298     /**
299      * Obtains an instance of {@code Instant} using milliseconds from the
300      * epoch of 1970-01-01T00:00:00Z.
301      * <p>
302      * The seconds and nanoseconds are extracted from the specified milliseconds.
303      *
304      * @param epochMilli  the number of milliseconds from 1970-01-01T00:00:00Z
305      * @return an instant, not null
306      * @throws DateTimeException if the instant exceeds the maximum or minimum instant
307      */
308     public static Instant ofEpochMilli(long epochMilli) {
309         long secs = Jdk8Methods.floorDiv(epochMilli, 1000);
310         int mos = Jdk8Methods.floorMod(epochMilli, 1000);
311         return create(secs, mos * 1000_000);
312     }

315     /**
316      * Obtains an instance of {@code Instant} from a temporal object.
317      * <p>
318      * A {@code TemporalAccessor} represents some form of date and time information.
319      * This factory converts the arbitrary temporal object to an instance of {@code Instant}.
320      * <p>
321      * The conversion extracts the {@link ChronoField#INSTANT_SECONDS INSTANT_SECONDS}
322      * and {@link ChronoField#NANO_OF_SECOND NANO_OF_SECOND} fields.
323      * <p>
324      * This method matches the signature of the functional interface {@link TemporalQuery}
325      * allowing it to be used as a query via method reference, {@code Instant::from}.
326      *
327      * @param temporal  the temporal object to convert, not null
328      * @return the instant, not null
329      * @throws DateTimeException if unable to convert to an {@code Instant}
330      */
331     public static Instant from(TemporalAccessor temporal) {
332         try {
333             long instantSecs = temporal.getLong(INSTANT_SECONDS);
334             int nanoOfSecond = temporal.get(NANO_OF_SECOND);
335             return Instant.ofEpochSecond(instantSecs, nanoOfSecond);
336         } catch (DateTimeException ex) {
337             throw new DateTimeException("Unable to obtain Instant from TemporalAccessor: " +
338                     temporal + ", type " + temporal.getClass().getName(), ex);
339         }
340     }

359     /**
360      * Obtains an instance of {@code Instant} using seconds and nanoseconds.
361      *
362      * @param seconds  the length of the duration in seconds
363      * @param nanoOfSecond  the nano-of-second, from 0 to 999,999,999
364      * @throws DateTimeException if the instant exceeds the maximum or minimum instant
365      */
366     private static Instant create(long seconds, int nanoOfSecond) {
367         if ((seconds | nanoOfSecond) == 0) {
368             return EPOCH;
369         }
370         if (seconds < MIN_SECOND || seconds > MAX_SECOND) {
371             throw new DateTimeException("Instant exceeds minimum or maximum instant");
372         }
373         return new Instant(seconds, nanoOfSecond);
374     }

390     /**
391      * Checks if the specified field is supported.
392      * <p>
393      * This checks if this instant can be queried for the specified field.
394      * If false, then calling the {@link #range(TemporalField) range} and
395      * {@link #get(TemporalField) get} methods will throw an exception.
396      * <p>
397      * If the field is a {@link ChronoField} then the query is implemented here.
398      * The supported fields are:
399      * <ul>
400      * <li>{@code NANO_OF_SECOND}
401      * <li>{@code MICRO_OF_SECOND}
402      * <li>{@code MILLI_OF_SECOND}
403      * <li>{@code INSTANT_SECONDS}
404      * </ul>
405      * All other {@code ChronoField} instances will return false.
406      * <p>
407      * If the field is not a {@code ChronoField}, then the result of this method
408      * is obtained by invoking {@code TemporalField.isSupportedBy(TemporalAccessor)}
409      * passing {@code this} as the argument.
410      * Whether the field is supported is determined by the field.
411      *
412      * @param field  the field to check, null returns false
413      * @return true if the field is supported on this instant, false if not
414      */
415     @Override
416     public boolean isSupported(TemporalField field) {
417         if (field instanceof ChronoField) {
418             return field == INSTANT_SECONDS || field == NANO_OF_SECOND || field == MICRO_OF_SECOND || field == MILLI_OF_SECOND;
419         }
420         return field != null && field.isSupportedBy(this);
421     }
422 
423     @Override
424     public boolean isSupported(TemporalUnit unit) {
425         if (unit instanceof ChronoUnit) {
426             return unit.isTimeBased() || unit == DAYS;
427         }
428         return unit != null && unit.isSupportedBy(this);
429     }

458     /**
459      * Gets the value of the specified field from this instant as an {@code int}.
460      * <p>
461      * This queries this instant for the value for the specified field.
462      * The returned value will always be within the valid range of values for the field.
463      * If it is not possible to return the value, because the field is not supported
464      * or for some other reason, an exception is thrown.
465      * <p>
466      * If the field is a {@link ChronoField} then the query is implemented here.
467      * The {@link #isSupported(TemporalField) supported fields} will return valid
468      * values based on this date-time, except {@code INSTANT_SECONDS} which is too
469      * large to fit in an {@code int} and throws a {@code DateTimeException}.
470      * All other {@code ChronoField} instances will throw a {@code DateTimeException}.
471      * <p>
472      * If the field is not a {@code ChronoField}, then the result of this method
473      * is obtained by invoking {@code TemporalField.getFrom(TemporalAccessor)}
474      * passing {@code this} as the argument. Whether the value can be obtained,
475      * and what the value represents, is determined by the field.
476      *
477      * @param field  the field to get, not null
478      * @return the value for the field
479      * @throws DateTimeException if a value for the field cannot be obtained
480      * @throws ArithmeticException if numeric overflow occurs
481      */
482     @Override  // override for Javadoc and performance
483     public int get(TemporalField field) {
484         if (field instanceof ChronoField) {
485             switch ((ChronoField) field) {
486                 case NANO_OF_SECOND: return nanos;
487                 case MICRO_OF_SECOND: return nanos / 1000;
488                 case MILLI_OF_SECOND: return nanos / 1000_000;
489                 case INSTANT_SECONDS: INSTANT_SECONDS.checkValidIntValue(seconds);
490             }
491             throw new UnsupportedTemporalTypeException("Unsupported field: " + field);
492         }
493         return range(field).checkValidIntValue(field.getFrom(this), field);
494     }

514      * @return the value for the field
515      * @throws DateTimeException if a value for the field cannot be obtained

528         }

583     /**
584      * Returns a copy of this instant with the specified field set to a new value.
585      * <p>
586      * This returns a new {@code Instant}, based on this one, with the value
587      * for the specified field changed.
588      * If it is not possible to set the value, because the field is not supported or for
589      * some other reason, an exception is thrown.
590      * <p>
591      * If the field is a {@link ChronoField} then the adjustment is implemented here.
592      * The supported fields behave as follows:
593      * <ul>
594      * <li>{@code NANO_OF_SECOND} -
595      *  Returns an {@code Instant} with the specified nano-of-second.
596      *  The epoch-second will be unchanged.
597      * <li>{@code MICRO_OF_SECOND} -
598      *  Returns an {@code Instant} with the nano-of-second replaced by the specified
599      *  micro-of-second multiplied by 1,000. The epoch-second will be unchanged.
600      * <li>{@code MILLI_OF_SECOND} -
601      *  Returns an {@code Instant} with the nano-of-second replaced by the specified
602      *  milli-of-second multiplied by 1,000,000. The epoch-second will be unchanged.
603      * <li>{@code INSTANT_SECONDS} -
604      *  Returns an {@code Instant} with the specified epoch-second.
605      *  The nano-of-second will be unchanged.
606      * </ul>
607      * <p>
608      * In all cases, if the new value is outside the valid range of values for the field
609      * then a {@code DateTimeException} will be thrown.
610      * <p>
611      * All other {@code ChronoField} instances will throw a {@code DateTimeException}.
612      * <p>
613      * If the field is not a {@code ChronoField}, then the result of this method
614      * is obtained by invoking {@code TemporalField.adjustInto(Temporal, long)}
615      * passing {@code this} as the argument. In this case, the field determines
616      * whether and how to adjust the instant.
617      * <p>
618      * This instance is immutable and unaffected by this method call.
619      *
620      * @param field  the field to set in the result, not null
621      * @param newValue  the new value of the field in the result
622      * @return an {@code Instant} based on {@code this} with the specified field set, not null
623      * @throws DateTimeException if the field cannot be set
624      * @throws ArithmeticException if numeric overflow occurs
625      */
626     @Override
627     public Instant with(TemporalField field, long newValue) {
628         if (field instanceof ChronoField) {
629             ChronoField f = (ChronoField) field;
630             f.checkValidValue(newValue);
631             switch (f) {
632                 case MILLI_OF_SECOND: {
633                     int nval = (int) newValue * 1000_000;
634                     return (nval != nanos ? create(seconds, nval) : this);
635                 }
636                 case MICRO_OF_SECOND: {
637                     int nval = (int) newValue * 1000;
638                     return (nval != nanos ? create(seconds, nval) : this);
639                 }
640                 case NANO_OF_SECOND: return (newValue != nanos ? create(seconds, (int) newValue) : this);
641                 case INSTANT_SECONDS: return (newValue != seconds ? create(newValue, nanos) : this);
642             }
643             throw new UnsupportedTemporalTypeException("Unsupported field: " + field);
644         }
645         return field.adjustInto(this, newValue);
646     }

649     /**
650      * Returns a copy of this {@code Instant} truncated to the specified unit.
651      * <p>
652      * Truncating the instant returns a copy of the original with fields
653      * smaller than the specified unit set to zero.
654      * The fields are calculated on the basis of using a UTC offset as seen
655      * in {@code toString}.
656      * For example, truncating with the {@link ChronoUnit#MINUTES MINUTES} unit will
657      * round down to the nearest minute, setting the seconds and nanoseconds to zero.
658      * <p>
659      * The unit must have a {@linkplain TemporalUnit#getDuration() duration}
660      * that divides into the length of a standard day without remainder.
661      * This includes all supplied time units on {@link ChronoUnit} and
662      * {@link ChronoUnit#DAYS DAYS}. Other units throw an exception.
663      * <p>
664      * This instance is immutable and unaffected by this method call.
665      *
666      * @param unit  the unit to truncate to, not null
667      * @return an {@code Instant} based on this instant with the time truncated, not null
668      * @throws DateTimeException if the unit is invalid for truncation
669      */
670     public Instant truncatedTo(TemporalUnit unit) {
671         if (unit == ChronoUnit.NANOS) {
672             return this;
673         }
674         Duration unitDur = unit.getDuration();
675         if (unitDur.getSeconds() > LocalTime.SECONDS_PER_DAY) {
676             throw new DateTimeException("Unit is too large to be used for truncation");
677         }
678         long dur = unitDur.toNanos();
679         if ((LocalTime.NANOS_PER_DAY % dur) != 0) {
680             throw new DateTimeException("Unit must divide into a standard day without remainder");
681         }
682         long nod = (seconds % LocalTime.SECONDS_PER_DAY) * LocalTime.NANOS_PER_SECOND + nanos;
683         long result = (nod / dur) * dur;
684         return plusNanos(result - nod);
685     }

765      * Returns a copy of this instant with the specified duration added.
766      * <p>
767      * This instance is immutable and unaffected by this method call.
768      *
769      * @param secondsToAdd  the seconds to add, positive or negative

772      * @throws DateTimeException if the result exceeds the maximum or minimum instant
773      * @throws ArithmeticException if numeric overflow occurs
774      */
775     private Instant plus(long secondsToAdd, long nanosToAdd) {
776         if ((secondsToAdd | nanosToAdd) == 0) {
777             return this;
778         }
779         long epochSec = Jdk8Methods.safeAdd(seconds, secondsToAdd);
780         epochSec = Jdk8Methods.safeAdd(epochSec, nanosToAdd / NANOS_PER_SECOND);

782         long nanoAdjustment = nanos + nanosToAdd;  // safe int+NANOS_PER_SECOND
783         return ofEpochSecond(epochSec, nanoAdjustment);
784     }

923     /**
924      * Calculates the period between this instant and another instant in
925      * terms of the specified unit.
926      * <p>
927      * This calculates the period between two instants in terms of a single unit.
928      * The start and end points are {@code this} and the specified instant.
929      * The result will be negative if the end is before the start.
930      * The calculation returns a whole number, representing the number of
931      * complete units between the two instants.
932      * The {@code Temporal} passed to this method is converted to a
933      * {@code Instant} using {@link #from(TemporalAccessor)}.
934      * For example, the period in days between two dates can be calculated
935      * using {@code startInstant.until(endInstant, SECONDS)}.
936      * <p>
937      * This method operates in association with {@link TemporalUnit#between}.
938      * The result of this method is a {@code long} representing the amount of
939      * the specified unit. By contrast, the result of {@code between} is an
940      * object that can be used directly in addition/subtraction:
941      * <pre>
942      *   long period = start.until(end, SECONDS);   // this method
943      *   dateTime.plus(SECONDS.between(start, end));      // use in plus/minus
944      * </pre>
945      * <p>
946      * The calculation is implemented in this method for {@link ChronoUnit}.
947      * The units {@code NANOS}, {@code MICROS}, {@code MILLIS}, {@code SECONDS},
948      * {@code MINUTES}, {@code HOURS}, {@code HALF_DAYS} and {@code DAYS}
949      * are supported. Other {@code ChronoUnit} values will throw an exception.
950      * <p>
951      * If the unit is not a {@code ChronoUnit}, then the result of this method
952      * is obtained by invoking {@code TemporalUnit.between(Temporal, Temporal)}
953      * passing {@code this} as the first argument and the input temporal as
954      * the second argument.
955      * <p>
956      * This instance is immutable and unaffected by this method call.
957      *
958      * @param endExclusive  the end date, which is converted to an {@code Instant}, not null
959      * @param unit  the unit to measure the period in, not null
960      * @return the amount of the period between this date and the end date
961      * @throws DateTimeException if the period cannot be calculated
962      * @throws ArithmeticException if numeric overflow occurs
963      */
964     @Override
965     public long until(Temporal endExclusive, TemporalUnit unit) {
966         Instant end = Instant.from(endExclusive);
967         if (unit instanceof ChronoUnit) {
968             ChronoUnit f = (ChronoUnit) unit;
969             switch (f) {
970                 case NANOS: return nanosUntil(end);
971                 case MICROS: return nanosUntil(end) / 1000;
972                 case MILLIS: return Jdk8Methods.safeSubtract(end.toEpochMilli(), toEpochMilli());
973                 case SECONDS: return secondsUntil(end);
974                 case MINUTES: return secondsUntil(end) / SECONDS_PER_MINUTE;
975                 case HOURS: return secondsUntil(end) / SECONDS_PER_HOUR;
976                 case HALF_DAYS: return secondsUntil(end) / (12 * SECONDS_PER_HOUR);
977                 case DAYS: return secondsUntil(end) / (SECONDS_PER_DAY);
978             }
979             throw new UnsupportedTemporalTypeException("Unsupported unit: " + unit);
980         }
981         return unit.between(this, end);
982     }
983 
984     private long nanosUntil(Instant end) {
985         long secsDiff = Jdk8Methods.safeSubtract(end.seconds, seconds);
986         long totalNanos = Jdk8Methods.safeMultiply(secsDiff, NANOS_PER_SECOND);
987         return Jdk8Methods.safeAdd(totalNanos, end.nanos - nanos);
988     }
989 
990     private long secondsUntil(Instant end) {
991         long secsDiff = Jdk8Methods.safeSubtract(end.seconds, seconds);
992         long nanosDiff = end.nanos - nanos;
993         if (secsDiff > 0 && nanosDiff < 0) {
994             secsDiff--;
995         } else if (secsDiff < 0 && nanosDiff > 0) {
996             secsDiff++;
997         }
998         return secsDiff;
999     }
